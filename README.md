# 新ヒヤリハット報告HTML（Claude作成）

建災防方式に準拠したヒヤリハット報告Webアプリの専用フォルダです。Claudeとのチャットで段階的に開発しています。

---

## ブラウザでの使い方

### 方法1：ローカルサーバーで起動（推奨）

1. **start.bat** をダブルクリックして実行
2. ブラウザが自動で開き、ポータル画面が表示されます
3. 「使用者」「管理者」「所有者」のいずれかをクリックして利用

※ Python または Node.js がインストールされている必要があります。

### 方法2：HTMLを直接開く

1. **index.html** をブラウザで開く（ダブルクリックまたはドラッグ＆ドロップ）
2. ポータル画面から各画面へ遷移

または、以下のファイルを直接開いても利用できます：
- `user.html` … 使用者（現場社員）
- `admin.html` … 管理者
- `owner.html` … 所有者

## 携帯・他PCからアクセスするには

**QRコードを有効にし、他端末からログインするには、HTTPSで公開されたURLが必要です。**

### 1. GitHub Pages にデプロイ（推奨）

1. このフォルダを GitHub リポジトリにプッシュ
2. リポジトリの **Settings → Pages** で Source を「main ブランチ」に設定
3. 公開URL例: `https://あなたのユーザー名.github.io/リポジトリ名/`
4. 携帯でこのURLを開くか、owner.html の設定タブでQRコードを表示して読み取り

### 2. Firebase でデータ共有（複数端末で同期）

1. [Firebase Console](https://console.firebase.google.com/) でプロジェクトを作成
2. Realtime Database を有効化
3. `config.example.js` を参考に `config.js` に設定を記入
4. 報告データ・管理者アカウントがクラウドで同期されます

### 3. ローカル開発時にQRコードを公開URL向けにする

`config.js` に以下を追加すると、ローカルで開いているときでもQRコードが公開URLを指します：

```javascript
window.HH_BASE_URL = "https://あなたのユーザー名.github.io/リポジトリ名/";
```

## 現在のファイル

| ファイル | 説明 |
|----------|------|
| `index.html` | ポータル画面（3つのHTMLへの入口） |
| `user.html` | 使用者用（報告入力・一覧・分析） |
| `admin.html` | 管理者用（一覧・分析・委員会） |
| `owner.html` | 所有者用（設定・QRコード・管理者登録） |
| `config.js` | Firebase設定（要編集） |
| `config.example.js` | 設定の記入例 |
| `hh-data.js` | 共有データ層（Firebase/localStorage） |
| `start.bat` | ローカルサーバー起動用（Windows） |
| `開発履歴・要件整理.md` | Claudeチャット履歴の整理・要件一覧 |

## 開発の経緯（概要）

- **目的**: ヒヤリハット報告の義務化・デジタル化により、現場の状況を管理職へ共有し、分析・改善につなげる
- **形式**: スマホ対応Webアプリ（URL共有で利用可能）
- **業種**: 建設・土木
- **ベース**: 建災防方式 新ヒヤリハット報告集計シート
