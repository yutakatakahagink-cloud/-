⚠ 複数端末で同一データを表示するには

携帯で報告した内容がPCに反映されない場合、Firebaseの設定が必要です。以下の手順で設定してください。

  1. Firebase Consoleでプロジェクトを作成
  2. Realtime Database を有効化（リージョン: asia-southeast1 シンガポール）
  3. プロジェクト設定 → 全般 → アプリを追加 → ウェブ → 設定値をコピー
  4. config.firebase.json を作成（config.firebase.json.example をコピーして値を貼り付け）
  5. config.firebase.json を GitHub にプッシュして反映

※ 全ての端末で同じURL（GitHub Pages等）から開いてください。file:/// やローカルでは同期されません。

📄 詳細手順（FirebaseConfig.md）

災害報告メールの送信元

提出・多段承認の通知に使う返信先（Reply-To）です。差戻し通知では、報告者宛メールに加え、このアドレスへ副本（BCC）も送ります（報告者メールと同じ場合は重複しないよう省略）。
専用メール1本＋EmailJSで運用する場合は、EmailJS に接続した送信用アドレスと同じをここにも入れ、config.js の fromEmail / replyToEmail と揃えると一貫します。
EmailJS 利用時はテンプレートに {{reply_to}} や {{sender_email}}、差戻し用に {{bcc_email}} を設定してください。

📱 QRコードについて：
各QRコードを印刷して配布してください。スキャンすると該当ページが開きます。
※ 報告項目などを修正してもQRコードは変わりません。印刷したQRコードはそのままご利用ください。
使用者：作業員が報告入力・一覧・分析を利用
管理者：ID/パスワードでログイン後、一覧・分析・委員会を利用
所有者：パスワードでログイン後、全機能を利用（設定で変更可能）

⚠ 注意：ローカルファイル（file:///）のQRコードはスマホでは開けません。
Webサーバー（GitHub Pages等）に配置したURLを使用してください。
📋 権限の説明：
所有者：管理者の追加・削除、全ページの閲覧が可能です。
管理者：一覧・分析（背後要因以外）・委員会ページを閲覧可能です。
使用者：報告ページと該当部署の分析（背後要因以外）を閲覧可能です。
